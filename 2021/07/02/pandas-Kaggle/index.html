Pandas索引

对于DataFrame的索引
有两种方式，一种像数组的索引，利用[][]来访问具体的某个entry，但是必须要清楚第一个[]表示列，第二个[]表示行。

另一种就是利用loc和iloc，假设df是一个DataFrame，df.loc[ a,b ]，a和b必须是分别是DataFrame的行标签和列标签。
df.iloc[a,b]，a和b分别是行号和列号。注意，利用loc或者iloc时，如果要取所有的列时，列可以省略，但是如果要取所有的行时，必须要加上 ‘：’，
也就是说，行一定不能省略

如果要去一段DataFrame的话，推荐使用loc或者iloc，不推荐使用数组索引
loc和iloc在切片索引上的区别：
loc由于是标签索引，所以在使用[a:b]，左闭右闭
iloc的[a:b]则是左闭右开

特别需要注意的是如果用了loc或者iloc，在[]内不能使用了’ : ‘，切片或者全选符号，

map和apply

它们都是用来转换DataFrame里的数据的作用，但是注意，它们都会返回一个DataFrame，但是不会改变使用这个方法的DataFrame
map是对于Series使用的，而apply是对于DataFrame。

map

对于map很明显是带入Series每一行的数据，然后返回一个Series。

apply

apply就需要注意
df.apply(function,axis=’columns’)，axis如果为’columns’，说明带入function的是df的row，需要改变的是某一整列
同理，如果axis=’index’，则带入function的是df每一列column，说明将会改变df某一整行，这种情况还是比较少见的，大部分情况还是改变一整列。

Summary function

对于Summary function，包括max,min,mean,median,std,unique,describe,value_counts等等..
这些函数都可以用在DataFrame的某一列，或者是Series

max,min,mean,median,std分别返回列的最大，最小，均值，中位数，标准差
unique:返回列中不重复的标签，例如列为[‘a’,’b’,’c’,’a’]，那么返回[‘a’,’b’,’c’]，返回的是一个numpy一维数列
value_counts:计算各个标签出现的次数，并且返回一个以标签作为index，次数作为value的Series

Groupby

个人认为，是通过columns标签对DataFrame进行分组，将标签相同的DataFrame的行，看成一个子DataFrame
然后对每个subDataFrame（只是个人理解，实际上没有）进行各种操作，将结果进行拼接，返回一个Series值，Series的index就是前面group不用的标签在DataFrame的value

进行一个操作返回的肯定是Series，可以应用agg([max,min,mean…])操作返回多个Summary function值，这时候返回的就是一个DataFrame

Sort

Method 1：sort_index
按照index的标签进行排序，ascending=True（缺省）,默认是按照从小到大排序
对于Series和DataFrame都是使用的

Method 2：sort_values（by=[]，ascending=True）
对于Series，由于没有columns名称，所以没有by参数
对于DataFrame，则需要by参数，例如by=[‘a’,’b’]，意味着，首先对a标签列进行从小到大排序，如果a相同的，则按照b进行从小到大排序。
